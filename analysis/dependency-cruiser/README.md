# Dependency-Cruiser Analysis Plan

## Selected tool

`dependency-cruiser` is a static analysis tool for JavaScript and Node.js projects. It inspects import/require relationships, validates them against custom architectural rules, and can emit text, JSON, and graph artifacts.

## Why it fits NodeBB

- NodeBB is a large JavaScript codebase with a clear split between server code in `src/` and browser bundle code in `public/src/`.
- The tool works directly on CommonJS-style dependency edges, which makes it a good fit for NodeBB without requiring a full type migration.
- It supports custom rules that can evolve with the project, which is useful for architecture drift detection over time.

## Customization tested in this branch

- Warn on circular dependencies in `src/` and `public/src/`.
- Error when server code imports browser bundle modules.
- Error when browser bundle code imports server core modules.
- Error when production code imports from `test/`.
- Exclude generated, vendor, upload, log, and upgrade-heavy paths to reduce noise.

## Reproduction steps

From the repository root:

```bash
npm --prefix install run analyze:dependency-cruiser
npm --prefix install run analyze:dependency-cruiser:json
npm --prefix install run analyze:dependency-cruiser:dot
```

To capture a terminal artifact for the pull request:

```bash
npm --prefix install run analyze:dependency-cruiser > analysis/dependency-cruiser/terminal-output.txt 2>&1
```

## Artifact checklist for the PR

- `install/package.json` shows the added dev dependency.
- `analysis/dependency-cruiser/npm-ls.txt` verifies that `dependency-cruiser` was installed locally.
- `install/package-lock.json` was also generated by npm, but this repository currently ignores `package-lock.json` files.
- `.dependency-cruiser.mjs` shows the project-specific configuration.
- `analysis/dependency-cruiser/terminal-output.txt` captures the CLI run.
- `analysis/dependency-cruiser/report.json` contains machine-readable findings.
- `analysis/dependency-cruiser/graph.dot` contains a graph artifact suitable for later visualization.

## Initial pros

- Easy npm-based installation in a JavaScript project.
- Low setup cost compared with heavier SAST platforms.
- Useful architectural feedback beyond style issues.
- Produces multiple artifact types that are easy to attach to a PR.

## Initial cons

- Reports dependency structure, not deeper dataflow or security issues.
- Rule tuning is necessary to avoid noise in a large legacy codebase.
- Large graphs become hard to interpret without collapsing or post-processing.
